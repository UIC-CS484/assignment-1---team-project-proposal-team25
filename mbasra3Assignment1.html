Project Tools and Requirements

  • This project will be using the Sakugabooru API
  • The API will provide direct access to search the image board to find, save, and study the posts they search for from the Sakugabooru website
  • Such data that could be collected would be what they search and what they save, in order to create statistics on what our demographic specifically is
  • username, email, password, post_saves, date_of_save, notes, note_modified_date (Still have to think this through)

Site Demographics and Goals

  • Our main demographic includes animation appreciators and those dedicated or aspiring to learn animation
  • Goal 1: To provide a space for learning animation without the need to download videos or run any external programs
  • Goal 2: To create a canvas designed to capture snippets of animation frame by frame and provide tools for marking and note-taking
  • Goal 3: To offer a basic guide on animation and what one should look out for when studying or creating an animation
  • Goal 4: To give users cloud space for saving, accessing, editing, and exporting their notes/studies
  • I wanted to make this application because as someone with a strong interest for animation and an appreciation for sakugabooru, I want to design an app that will allow us to save our notes and studies in one space, like a notebook
